PROJECT PHILOSOPHY
==========================================================

This project is NOT a desktop application.

This project is NOT a migration.

This project is a completely new, modern web application inspired by an existing business workflow.

The objective is NOT to copy old code.

The objective is to build the best possible implementation while preserving the business logic.

Always prioritize:

• Clean Architecture
• Maintainability
• Simplicity
• Performance
• Scalability
• Security
• Excellent User Experience

Never introduce unnecessary complexity.

==========================================================
BUSINESS RULES
==========================================================

Never violate the business rules defined in the documentation.

Examples include:

Order Amount = Quantity × Rate

Transport Amount = Quantity × Rate

Remaining Quantity = Total Quantity − Dispatched Quantity

Hissab Amount = Dispatched Quantity × Rate

Dispatch quantity must never exceed remaining quantity.

Statuses must follow valid transitions.
